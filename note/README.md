# LoRR2026 Notes

`note/` 用于沉淀比赛理解、代码框架、可行方法、论文阅读与工程约束。

## 目录骨架

- `note.md`
  - 比赛设定、输入输出、系统主循环、执行语义的总览笔记。
- `alg.md`
  - starter-kit 代码结构、关键类、调用链、默认算法入口整理。
- `method.md`
  - 可落地的方法路线、分赛道改造方向、优先级建议。
- `paper.md`
  - 与 LoRR2026 / Lifelong MAPF 直接相关的论文清单与阅读顺序。
- `tech.md`
  - 时间预算、运行约束、调参时必须盯住的系统级细节。
- `skeleton.md`
  - 在“不改受保护文件”前提下，三赛道可共用的本地开发骨架与实现落点。

## 建议阅读顺序

1. 先看 `note.md`，建立题目与系统整体认知。
2. 再看 `alg.md`，把代码入口、数据流和默认模块串起来。
3. 接着看 `tech.md`，明确时间预算、慢环 / 快环和执行限制。
4. 然后看 `method.md`，决定当前迭代到底优化什么。
5. 需要补背景时看 `paper.md`。
6. 准备真正动手实现时看 `skeleton.md`。

## 维护约定

- 以“能支持下一轮实现和复盘”为目标，不写空泛总结。
- 优先记录可验证结论、代码位置、输入输出约束和样例表现。
- 需要新增专题时，尽量新建独立文档，不把 `note.md` 堆成杂项仓库。
